# 自动认证与数据采集样例实现记录

## 1. 实现位置

样例代码位于：

`examples/browser-data-collector`

## 2. 已实现能力

- 模拟内部统一认证页面。
- 支持已有有效认证状态。
- 支持短期认证失效后的快速认证按钮点击。
- 支持长期认证失效后的指纹认证等待状态。
- 指纹认证场景产生内部 IM 提醒模型。
- 支持进入目标页面后的子页面点击。
- 支持下拉框操作。
- 支持点击查询并等待关键响应。
- 记录请求方法、URL、请求体和响应状态。
- 按配置提取嵌套 JSON 字段。
- 输出标准化数据记录。
- 保存认证、操作、请求和结果审计轨迹。
- 支持真实页面有头探测模式。
- 支持人工认证和人工页面操作记录。
- 支持 XHR、Fetch、JSON 请求候选筛选和评分。
- 支持敏感请求信息和 JSON 敏感字段遮蔽。
- 支持查看候选响应的 JSON 字段路径。
- 支持浏览器认证后使用共享会话发送纯 HTTP 请求。
- 支持数据源、业务报告、消息通道和工作流分层。
- 支持生成 Markdown 报告和 SVG 图表。
- 支持 JSON Webhook 消息适配器。
- 支持每日定时调度。

## 3. 配置化内容

以下内容在 `config/data-source.json` 中配置：

- 数据源 ID 和名称。
- 入口路径。
- 目标页面 URL 特征。
- 认证页面 URL 特征。
- 快速认证按钮。
- 指纹认证按钮。
- 页面操作步骤。
- 关键响应 URL 特征。
- 记录集合路径。
- 字段映射。
- 主键。

## 4. 验证结果

端到端测试已覆盖三种场景：

| 场景 | 结果 |
| --- | --- |
| 已有有效认证 | 成功进入目标页并采集 2 条记录 |
| 短期认证失效 | 自动点击快速认证，成功采集 2 条记录 |
| 长期认证失效 | 产生 IM 模拟提醒，模拟指纹完成后采集 2 条记录 |

运行命令：

```bash
npm test
```

下一步优先把样例连接到一个可控的真实网页或内部脱敏样例，并建设“配置工作台”：

1. 内网使用真实页面探测器完成第一轮验证。
2. 根据反馈完善页面操作生成规则。
3. 建设图形化候选请求列表。
4. 展示响应 JSON 树和表格预览。
5. 由用户选择记录节点和字段。
6. 生成数据源配置。
7. 一键试跑并保存新版本。
