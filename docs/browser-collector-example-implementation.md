# 自动认证与数据采集样例实现记录

## 1. 实现位置

样例代码位于：

`examples/browser-data-collector`

## 2. 已实现能力

- 模拟内部统一认证页面。
- 支持已有有效认证状态。
- 支持短期认证失效后的快速认证按钮点击。
- 支持长期认证失效后的指纹认证等待状态。
- 指纹认证场景产生内部 IM 提醒模型。
- 支持进入目标页面后的子页面点击。
- 支持下拉框操作。
- 支持点击查询并等待关键响应。
- 记录请求方法、URL、请求体和响应状态。
- 按配置提取嵌套 JSON 字段。
- 输出标准化数据记录。
- 保存认证、操作、请求和结果审计轨迹。

## 3. 配置化内容

以下内容在 `config/data-source.json` 中配置：

- 数据源 ID 和名称。
- 入口路径。
- 目标页面 URL 特征。
- 认证页面 URL 特征。
- 快速认证按钮。
- 指纹认证按钮。
- 页面操作步骤。
- 关键响应 URL 特征。
- 记录集合路径。
- 字段映射。
- 主键。

## 4. 验证结果

端到端测试已覆盖三种场景：

| 场景 | 结果 |
| --- | --- |
| 已有有效认证 | 成功进入目标页并采集 2 条记录 |
| 短期认证失效 | 自动点击快速认证，成功采集 2 条记录 |
| 长期认证失效 | 产生 IM 模拟提醒，模拟指纹完成后采集 2 条记录 |

运行命令：

```bash
npm test
```

## 6. 下一步建议

下一步优先把样例连接到一个可控的真实网页或内部脱敏样例，并建设“配置工作台”：

1. 打开有头浏览器。
2. 记录用户页面操作。
3. 展示操作期间产生的请求列表。
4. 由用户选择关键请求。
5. 展示响应 JSON 树和表格预览。
6. 由用户选择记录节点和字段。
7. 生成数据源配置。
8. 一键试跑并保存新版本。
