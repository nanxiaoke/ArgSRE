# 高阶云服务 L1 接管卡片填写模板：以华为云 DWS / DMS 为样例

> 目的：用于手工补齐 L1 基础运维能力。当前阶段不追求完整白皮书，而是先保证“有事不乱、升级有效、信息完整、边界清楚、风险可见”。
>
> 注意：DWS / DMS 只是样例，不应把平台模型绑定到这两个服务。平台需要支持后续扩展到所有高阶云服务。

## 1. 设计原则：通用模型优先，服务类型扩展

L1 接管卡片应分为两层：

1. 通用接管卡片

   所有高阶云服务都必须填写，用于表达服务责任、入口、最小画像、观测、升级、只读检查、操作边界和缺口行动项。

2. 服务类型扩展

   不同服务类型按需扩展。例如数据库/数据仓库类关注集群、节点、连接入口和备份；消息队列类关注 Topic、消费组、堆积和容量策略；容器类关注集群、节点池、工作负载和调度；缓存类关注实例、分片、内存、连接数和淘汰策略。

平台落地时，应把通用字段作为主模型，把 DWS / DMS 等服务特有字段做成扩展字段或类型插件，避免后续推广时重构。

### 1.1 通用对象抽象

为了支持所有高阶服务，建议抽象以下对象：

- 服务：一个云服务或一类云产品，例如 DWS、DMS、CCE、RDS、Redis。
- 服务实例：一个具体实例、集群、资源池或命名空间。
- 管理面：控制台、API、调度器、任务中心、元数据服务等。
- 数据面：实际承载用户流量或数据的实例、节点、代理、分片、Broker、Pod 等。
- 业务对象：服务内部关键对象，例如数据库、表、Topic、消费组、实例、工作负载、租户。
- 接入入口：控制台、API、客户端连接地址、ELB、域名、内网地址、公网地址。
- 观测入口：监控、告警、日志、事件、拨测、审计。
- 操作边界：L1 可做的只读检查、禁止操作、需升级操作。
- 数据源绑定：平台采集或人工导入该服务信息的数据来源。

| 服务类型 | 典型服务 | 扩展对象 |
| --- | --- | --- |

## 2. 官网资料对 DWS / DMS 样例的启发

### 2.1 DWS 服务特征

DWS 是数据仓库服务，官网定位为全场景数据仓库和一站式分析平台。公开资料中强调集群规模扩展、分布式并行计算、多层级 HA、备份、硬件冗余、弹性扩容、仓湖一体、OBS 数据访问、多源数据融合和实时分析。

对 L1 接管来说，DWS 至少要关注：

- 集群在哪里：Region、AZ、VPC、子网、安全组。
- 集群是什么形态：存算一体或存算分离、节点规格、节点数量、版本。
- 入口是什么：控制台、集群列表、数据库连接端口、ELB、监控、日志、告警。
- 高可用底线是什么：是否多 AZ、是否配置 ELB、是否避免业务直连单 CN。
- 风险边界是什么：扩容、规格变更、安全组变更、公网访问、备份恢复等都不是 L1 随意操作项。

### 2.2 DMS / Kafka 服务特征

DMS 是分布式消息服务，官网定位为兼容 Kafka、RocketMQ、RabbitMQ 的高可用消息队列服务。Kafka 版强调高吞吐、高可用、托管、兼容开源 Kafka、Topic、分区、副本、消息堆积、SASL/SSL 接入、云审计和多 AZ。

对 L1 接管来说，DMS/Kafka 至少要关注：

- 实例在哪里：Region、AZ、VPC、子网、安全组。
- 实例是什么形态：Kafka/RocketMQ/RabbitMQ，Kafka 版本，单机或集群，代理规格和代理数量。
- 业务对象是什么：Topic、分区、副本、消费组、生产者、消费者。
- 运行态看什么：生产流量、消费流量、消息堆积、磁盘使用率、连接数、代理状态、消费延迟。
- 高风险边界是什么：公网访问、安全协议、Topic 自动创建、容量阈值策略、扩容、删除 Topic、修改老化时间等。

## 7. 后续扩展方式

当推广到新的高阶服务时，不应复制整份模板，而应按以下步骤扩展：

1. 复用通用 L1 服务接管卡片。
2. 判断服务类型，例如数据库、消息、容器、缓存、网络、存储、大数据。
3. 选择或新增服务类型扩展字段。
4. 补充该服务的关键业务对象、关键观测项和高风险边界。
5. 将新增字段沉淀为类型模板，供同类服务复用。

平台模型上建议采用：

- `ServiceProfile`：通用服务画像。
- `ServiceInstance`：服务实例或集群。
- `ServiceObject`：服务内部业务对象。
- `ObservationBinding`：监控、告警、日志、审计等入口。
- `OperationBoundary`：只读检查、禁止操作、需升级操作。
- `TypeExtension`：DWS、DMS、CCE 等服务类型扩展字段。
