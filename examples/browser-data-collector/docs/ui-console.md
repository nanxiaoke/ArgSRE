# ArgSRE 数据采集控制台

这个控制台把前期命令行验证流程 UI 化，用于内网常态化录入认证入口、页面探测、候选请求选择、字段提取配置、重放验证和报告查看。

## 一键启动

```bash
cd examples/browser-data-collector
npm run ui
```

默认地址是 `http://127.0.0.1:8787`。如果端口被占用，可以指定：

```bash
ARGSRE_UI_PORT=8788 npm run ui
```

## 架构边界

- `src/core`：采集、解析、质量规则、历史、报告等核心能力。
- `src/probe.js`：真实浏览器探测能力，支持 UI 主动结束探测。
- `src/ui-server.js`：本机 HTTP API，只编排已有核心能力，不保存敏感数据到代码目录。
- `src/ui`：静态前端页面，负责配置编辑、触发执行、查看本机结果。
- `runtime/local-config`：本机私有配置，受 `.gitignore` 保护。
- `runtime/daily-reports` 和 `runtime/probes`：本机运行产物，受 `.gitignore` 保护。

## 当前 UI 能力

- 数据源配置：编辑入口 URL、认证按钮、请求 URL、请求体、字段映射、质量规则。
- 配置管理：保存和读取 `runtime/local-config/*.json`。
- 配置校验：调用与 CLI 相同的配置校验逻辑。
- 页面探测：打开真实 Edge 浏览器，记录操作和候选 XHR/fetch 请求。
- 探测控制：启动后可在 UI 点击“结束探测”，不再依赖命令行 Enter。
- 候选选择：在 UI 里选择候选请求，并回填请求方法、URL、请求体到数据源配置。
- 字段提取：根据候选响应展示 JSON 路径，选择 `recordPath` 并逐个加入字段映射。
- 重放验证：不生成报告，直接使用当前数据源配置发起一次采集验证，查看记录数、样例和审计。
- 报告生成：配置验证通过后可生成本机 dry-run 报告，消息发送落本地文件。
- 结果查看：查看最近执行审计、最近探测摘要和最新 Markdown 报告。

## 推荐闭环流程

1. 启动 UI，保存一个 `daily-report.local.json`。
2. 在“页面探测”里输入内部入口 URL，设置较长自动结束时间，例如 3600 秒。
3. 点击“开始探测”，在弹出的浏览器中完成认证、进入目标子页面、选择下拉框并触发查询。
4. 回到 UI 点击“结束探测”。
5. 在“候选请求”里选择目标 XHR/fetch 请求。
6. 点击“应用到配置”，把请求方法、URL、请求体回填到数据源配置。
7. 在“推荐数据路径”中选择记录数组路径，点击“使用选中数据路径”。
8. 在“响应字段路径”中选择字段，填写输出字段名，点击“加入字段映射”。
9. 回到“数据源配置”，点击“校验”。
10. 点击“重放验证”，确认记录数、样例字段和审计状态符合预期。
11. 点击“生成报告”，查看 `workflow-audit` 和 `report.md`。

## 反馈约束

不要把 `runtime` 目录中的配置、截图、请求、响应、报告原文发到外网。反馈时只发送脱敏后的状态、错误码、候选数量、记录数量、失败分类和必要的抽象字段路径。
